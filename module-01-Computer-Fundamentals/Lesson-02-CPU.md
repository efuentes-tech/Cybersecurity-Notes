# Lesson 2 - CPU

## Objective 
Understanding what the CPU actually does.

## CPU Cycle:
Fetch -> Decode -> Execute

## What I Learned
Today I learned that the CPU is responsible for executing instructions. For example, when I open a program, the operating system loads it from storage into RAM and the CPU continuously fetches, decodes, and executes the program's instructions. It performs billions of these intructions every second which allows applications and the operating system to function. 

## Troubleshooting Scenarion
A user says "My computer is really slow."
You now know to ask yourself:
Is the RAM full?
Is the SSD nearly full? 
Is another program using the CPU?
Is the CPU overheating?
Is malware consuming resources? 

## Why this matters?
This matters because when troubleshooting, you know where the work is actually happening. Storage stores, RAM holds active data, but the CPU performs the work. This is important when diagnosing performance problems.
